The ledger

The Heresy Ledger currently records no executable non-Fortran code. Every program in this repository — the site generator, the build operator, the validators, the tests — is Fortran. The measurement above is enforced twice: by the test suite, and by a validator that refuses any generated page containing a script tag.

The accounting is honest about its boundaries. These are platform tools, not heresy, and this site does not pretend otherwise:

  • GFortran, which compiles the Cathedral;
  • fpm, the Fortran package manager, which orders the build;
  • Git and the GitHub CLI, which Forty directs but never impersonates;
  • cmd.exe, through which Fortran addresses the operating system;
  • and your browser, which renders what Fortran wrote.

HTML, CSS, SVG, XML, and JSON in this site are declarative artifacts, all emitted by Fortran. They are output, not code.

The pipeline

There is no framework behind this page. A route registry in Fortran declares each page; a Fortran module assembles head, navigation, and body; escaping passes every special character through a Fortran function; and the writer lays the files into the porch directory. The stylesheet, its design tokens, the rose window (placed by sine and cosine), the sitemap, the robots.txt, and the route manifest are emitted by the same hand. The verger Forty drives all of it: forty generate, forty validate, forty test.

The source is public: github.com/teamofoxen/fortran-cathedral.

The mandate

The governing rule of this project, quoted without apology: use Fortran for every component it can plausibly perform safely, including trivial, awkward, ceremonial, infrastructural, and comically inappropriate work. When two approaches are viable, prefer the one that gives Fortran the more unnecessary job.
