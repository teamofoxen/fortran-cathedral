The Hall of Deprecated Syntax

Languages retire their past in different ways, and flattening them all into “deprecated” teaches nothing. This hall keeps six exhibits, each labeled with its precise fate. Five fates are possible:

Currently standard
In the present standard, unretired — however unfashionable.
Obsolescent
Still standard, formally discouraged; better means exist and removal may come.
Deleted
Removed from a named edition of the standard. Compilers may still accept it for the old scriptures.
Never standard, historically common
Vendor custom so widespread it was mistaken for the standard.
Compiler extension
Accepted by a compiler beyond the standard, documented as such.

Every classification below is cited, and every scroll carries the compiler's own testimony — measured by this site's generator invoking GFortran at build time, under -std=f2018 and in its merciful default, then measured again by the validator. The Cathedral does not assert what it can probe.

The Arithmetic IF

One test, three destinations: negative, zero, or positive, each named by a statement label. On the IBM 704, whose hardware compared against zero in a single breath, this was the natural shape of a branch.

Classification
Deleted — declared obsolescent in Fortran 90 and deleted outright in Fortran 2018[1].
The modern preference
The block IF: if … else if … else, which says the three cases in words.
The compiler's testimony
REJECTED UNDER -std=f2018; ACCEPTED AS AN EXTENSION BY DEFAULT — Error: Fortran 2018 deleted feature: Arithmetic IF statement

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.

Hollerith Constants

Characters counted into numeric storage — 4HHELO is the four characters HELO packed into an integer — from the age before Fortran had a character type at all.

Classification
Deleted — Fortran 66 practice removed from the FORTRAN 77 standard (kept only in an appendix); the related H edit descriptor lingered until its deletion in Fortran 95[2]. Lives on as a documented compiler extension[4].
The modern preference
The character type, standard since FORTRAN 77: characters live in character variables.
The compiler's testimony
REJECTED UNDER -std=f2018; ACCEPTED AS AN EXTENSION BY DEFAULT — Error: Legacy Extension: Hollerith constant

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.

The Computed GO TO

Branch by position: an integer selects the k-th label from a list. A jump table written by hand, and the ancestor of every dispatch switch.

Classification
Obsolescent — since Fortran 95, and still in the standard today[2].
The modern preference
The select case construct, which names its branches and needs no labels[2].
The compiler's testimony
ACCEPTED WITH WARNING UNDER -std=f2018 — Warning: Obsolescent feature: Computed GOTO

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.

The Statement Function

A one-line function defined among the declarations — compact, convenient, and subject to famously nonintuitive restrictions.

Classification
Obsolescent — since Fortran 95; wholly superseded[2].
The modern preference
The internal function after contains: checked, typed, and free of the old restrictions[2][3].
The compiler's testimony
ACCEPTED WITH WARNING UNDER -std=f2018 — Warning: Obsolescent feature: Statement function

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.

The Plain GO TO

The unconditional jump: control moves to the named label, no questions asked. Every loop in this Hall's era was built from it.

Classification
Currently standard — the unconditional GO TO has never been declared obsolescent or deleted; no standards source lists it among the retired[6]. Style retired it; the standard did not. The Hall includes it precisely so “old” is not mistaken for “deprecated”.
The modern preference
Structured constructs — do, exit, cycle, select case — wherever they say the intent better.
The compiler's testimony
ACCEPTED CLEANLY UNDER -std=f2018

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.

REAL*8

Precision by byte count, in the vendor dialects of the seventies — so widespread that generations mistook it for scripture.

Classification
Never standard — a vendor custom in no edition of the standard, carried today as a documented compiler extension[5].
The modern preference
Kind parameters: selected_real_kind (Fortran 90) or the named constants of iso_fortran_env (Fortran 2008).
The compiler's testimony
REJECTED UNDER -std=f2018; ACCEPTED AS AN EXTENSION BY DEFAULT — Error: GNU Extension: Nonstandard type declaration REAL*8

The modern reading's testimony: ACCEPTED CLEANLY UNDER -std=f2018.
