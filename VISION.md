# Vision — Fortran Cathedral

## The grand idea

Fortran Cathedral is a shrine to the oldest language in the room that still has work to do.

It honors the strange fact that a language born in the 1950s still sits beneath weather models, physics codes, engineering simulations, scientific libraries, and supercomputers — often unseen by the people benefiting from it.

The site should feel like entering a sacred machine room.

Not a museum of a dead language.

A cathedral built around an engine that is still running.

## The second grand idea

The site must not merely explain Fortran's durability. It must demonstrate Fortran's durability by assigning it jobs that are far beneath, far outside, and occasionally hostile to its dignity.

Fortran will not only generate the serious numerical history pages.

Fortran should also, whenever safely possible:

- decide which navigation item is active,
- emit the favicon,
- count its own impurities,
- generate `robots.txt`,
- validate content files,
- stamp releases,
- check links,
- print local setup instructions,
- prepare deployment artifacts,
- initialize the repository,
- and escort the project to GitHub through Forty.

This excess is not incidental. It is the art.

## The experience

A visitor should immediately understand:

- Fortran is old.
- Fortran is not merely obsolete.
- Modern Fortran is different from the stereotype.
- Numerical computing still owes it an enormous debt.
- The site itself is proof that Fortran can be pushed absurdly far outside its natural habitat.
- The builders did not abandon the premise when it became inconvenient.

The humor comes from architectural commitment, not cheap mockery.

## The architecture as exhibit

The repository, build process, status output, and deployment story are part of the public work.

Visitors should be able to inspect:

- what Fortran generated,
- what Forty orchestrated,
- what non-Fortran code remains,
- why each impurity exists,
- and how many lines of heresy survived the latest build.

The architecture should feel like another wing of the Cathedral, not backstage clutter.

## Forty

Forty is the Cathedral's Fortran operator: part build tool, part curator, part solemnly overqualified church sexton.

Forty should become the ordinary way to build, test, inspect, connect, and prepare the Cathedral.

When the repository first meets GitHub, Forty should make the introduction.

Forty may rely on Git and the official GitHub CLI for platform and credential boundaries, but the sequence, checks, prompts, verification, and final proclamation belong to Fortran.

Expected final message:

> `CONNECTION CONSECRATED. THE REMOTE IS CANONICAL.`

## Possible site sections

### The Nave

The main landing experience: the thesis, the visual identity, and the explanation of why this site exists.

### Why It Still Stands

A clear explanation of Fortran's survival: performance, compilers, arrays, validated scientific code, institutional investment, and ecosystem depth.

### Old Testament / Modern Testament

Side-by-side examples of fixed-form legacy Fortran and modern free-form Fortran.

### The Book of BLAS

A guided explanation of BLAS, LAPACK, and the numerical foundations hidden beneath modern scientific software.

### Saints of Numerical Computing

Profiles of people, projects, compilers, libraries, and institutions that shaped numerical computing.

### Hall of Deprecated Syntax

A loving tour through historical oddities, punched-card constraints, labels, COMMON blocks, and other artifacts.

### The Reliquary

A gallery of source listings, manuals, diagrams, code excerpts, compiler output, and architecture artifacts.

### The Confessional

A public accounting of every non-Fortran executable component, why it exists, how many lines it contains, and whether Forty believes it can be purged.

### The Scriptorium

Interactive or simulated exhibits: code conversion, syntax exploration, compiler demonstrations, or generated examples.

### The Verger's Room

A live or generated view of Forty: build status, compiler details, route count, heresy count, Git state, and declarations of numerical dignity.

## Voice

The site speaks with reverence, precision, and dry humor.

It does not sneer at Fortran.
It does not pretend Fortran is ideal for everything.
It does not conceal the absurdity of using it for the web.
It does not abandon the premise when implementation becomes inconvenient.
It never winks so hard that the joke collapses.
