Why it still stands

Fortran was first delivered by IBM in 1957, from a team led by John Backus, as a way to write mathematics for machines without writing assembly. It is not a museum piece. The language remains a living ISO standard — most recently Fortran 2023 — with modern modules, explicit interfaces, allocatable arrays, and whole-array arithmetic.

Much of numerical computing still rests on it. The reference implementations of BLAS and LAPACK — the linear-algebra foundations beneath a great deal of modern scientific software — are Fortran. Major operational weather models, including ECMWF's IFS, are written largely in Fortran. When the forecast arrives, an old and disciplined language helped carry it.

Fortran endures for unromantic reasons: arrays are first-class citizens, aliasing rules favor aggressive compiler optimization, decades of validated numerical code exist and still compile, and institutions do not casually rewrite the software that predicts hurricanes.

The verger's hand

The exhibit below is not a mock-up. It is a working function quoted verbatim from this site's own build tool, Forty — modern, free-form Fortran, escaped for your protection by another Fortran function:

!> Strip trailing carriage returns and trailing blanks.
!> Captured command output on Windows deserves this courtesy.
pure function trim_cr(s) result(r)
  character(*), intent(in) :: s
  character(:), allocatable :: r
  integer :: n
  n = len_trim(s)
  do while (n > 0)
    if (s(n:n) == achar(13)) then
      n = n - 1
    else
      exit
    end if
  end do
  r = s(1:n)
end function trim_cr
